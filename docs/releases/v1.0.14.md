# v1.0.14

## Changes

更新内容:

- 修复 Web Docker 镜像发布产物校验问题，避免镜像启动后立即退出。
- 桌面端新增内置更新能力，并在关于页提供版本号展示和检查更新入口。
- Web 端支持检查容器镜像最新版本，和桌面端更新策略保持一致。
- macOS 增强打开能力，支持在访达、终端以及常见第三方终端中打开相关目录或命令。
- 新增关于页 Tab，并统一更新入口、来源选择和 provider 下拉控件的交互样式。
- 完善发布流程，强制校验更新签名、发布说明和 release 产物，降低漏发或错发风险。
- 补充通用内置更新方案文档，便于后续复用同类更新策略。

Updates:

- Fixed Web Docker image artifact validation to avoid containers exiting immediately after startup.
- Added built-in desktop update support with version display and update checks in the About tab.
- Added Web container version checks aligned with the desktop update strategy.
- Improved macOS open actions for Finder, Terminal, and common third-party terminals.
- Added an About tab and aligned update entry, source selector, and provider dropdown styling.
- Hardened the release pipeline with updater signature, release notes, and artifact validation.
- Added a generic built-in update strategy document for reuse in similar applications.
